# 音悦音频播放器 - 开发文档

## 一、需求分析

### 1.1 产品定位

音悦音频播放器是一款基于 Java 开发的桌面端音频播放软件，致力于为用户提供简洁、高效、美观的音乐播放体验。软件支持多种音频格式，提供丰富的播放控制功能和个性化设置。

### 1.2 目标用户

- 音乐爱好者
- 音频工作者
- 需要本地音乐管理的普通用户

- 启动时间 < 3秒
- 播放响应延迟 < 100ms
- 内存占用 < 200MB（正常播放时）
- 支持播放列表 1000+ 首歌曲

- 支持 macOS 10.15+
- 支持 Linux（Ubuntu 18.04+）

- 异常情况有友好提示

## 三、系统架构设计

### 3.1 架构模式

采用 **MVC（Model-View-Controller）** 架构模式，结合分层设计思想，确保系统的可维护性和可扩展性。

```
┌─────────────────────────────────────────────────┐
│                   View 视图层                    │
│  (JavaFX UI / FXML / CSS)                       │
│  MainView, PlayerControlBar, PlaylistView...    │
├─────────────────────────────────────────────────┤
│                 Controller 控制层               │
│  MainController, PlayerController...            │
│  (处理用户交互，协调View与Service)               │
├─────────────────────────────────────────────────┤
│                  Service 服务层                 │
│  AudioPlayerService, PlaylistService...         │
│  (核心业务逻辑)                                  │
├─────────────────────────────────────────────────┤
│                   Model 模型层                  │
│  Song, Playlist, AudioInfo...                   │
│  (数据模型与状态管理)                            │
├─────────────────────────────────────────────────┤
│                    Util 工具层                  │
│  AudioUtils, FileUtils, LrcParser...            │
│  (通用工具与底层封装)                            │
└─────────────────────────────────────────────────┘
```

### 3.2 模块划分

#### 3.2.1 音频播放模块 (AudioPlayerService)

**职责**：负责音频文件的解码、播放、暂停、停止等核心播放控制。

**核心接口**：
```java
public interface AudioPlayerService {
    void play(Song song);           // 播放指定歌曲
    void pause();                   // 暂停
    void resume();                  // 恢复播放
    void stop();                    // 停止
    void seek(long position);       // 跳转到指定位置
    void setVolume(float volume);   // 设置音量
    void setMute(boolean mute);     // 设置静音
    boolean isPlaying();            // 是否正在播放
    long getCurrentPosition();      // 获取当前播放位置
    long getDuration();             // 获取总时长
}
```

**设计要点**：
- 使用策略模式支持不同音频格式的解码器
- 采用观察者模式通知播放状态变化
- 后台线程处理音频解码，不阻塞UI线程

#### 3.2.2 播放列表模块 (PlaylistService)

**职责**：管理播放列表的增删改查，播放模式控制，歌曲切换逻辑。

**核心接口**：
```java
public interface PlaylistService {
    void addSong(Song song);                // 添加歌曲
    void removeSong(int index);             // 移除歌曲
    void playSong(int index);               // 播放指定位置歌曲
    void playNext();                        // 下一曲
    void playPrevious();                    // 上一曲
    void setPlayMode(PlayMode mode);        // 设置播放模式
    PlayMode getPlayMode();                 // 获取播放模式
    List<Song> getPlaylist();               // 获取播放列表
    int getCurrentIndex();                  // 获取当前播放索引
    void savePlaylist(String name);         // 保存播放列表
    void loadPlaylist(String name);         // 加载播放列表
}
```

**播放模式枚举**：
```java
public enum PlayMode {
    SEQUENCE,       // 顺序播放
    REPEAT_ONE,     // 单曲循环
    REPEAT_ALL,     // 列表循环
    SHUFFLE         // 随机播放
}
```

#### 3.2.3 音乐库模块 (LibraryService)

**职责**：本地音乐文件扫描、索引管理、分类浏览。

**核心接口**：
```java
public interface LibraryService {
    void scanDirectory(String path);        // 扫描指定目录
    List<Song> searchSongs(String keyword); // 搜索歌曲
    List<String> getAllArtists();           // 获取所有歌手
    List<Song> getSongsByArtist(String artist);  // 按歌手查找
    List<String> getAllAlbums();            // 获取所有专辑
    List<Song> getSongsByAlbum(String album);    // 按专辑查找
    void refreshLibrary();                  // 刷新音乐库
}
```

#### 3.2.4 音效处理模块 (AudioEffectService)

**职责**：均衡器、混响、变速等音效处理。

**核心接口**：
```java
public interface AudioEffectService {
    void setEqualizer(float[] bands);       // 设置均衡器
    float[] getEqualizer();                 // 获取均衡器设置
    void setPlaybackSpeed(float speed);     // 设置播放速度
    float getPlaybackSpeed();               // 获取播放速度
    void setPitch(float pitch);             // 设置音调
    float getPitch();                       // 获取音调
    void setReverb(boolean enable);         // 设置混响开关
}
```

### 3.4 事件机制

采用事件驱动架构，各模块之间通过事件解耦。

#### 核心事件类型

| 事件类型 | 触发时机 | 携带数据 |
|---------|----------|----------|
| PlayStateChangedEvent | 播放状态变化 | 当前状态、歌曲信息 |
| ProgressChangedEvent | 播放进度更新 | 当前位置、总时长 |

| PlaylistChangedEvent | 播放列表变更 | 变更类型、索引 |
| SongChangedEvent | 当前歌曲切换 | 新歌曲信息 |

## 四、关键技术方案

### 4.1 音频播放引擎设计

#### 4.1.1 架构设计

音频播放引擎采用分层设计，分为解码器层、音频处理层、输出层。

```
音频文件 → 解码器 → 音频处理（EQ/变速等） → 音频输出（SourceDataLine）
```

#### 4.1.2 解码器策略

使用策略模式封装不同格式的解码器：

```java
public interface AudioDecoder {
    boolean supports(String format);
    AudioInfo getAudioInfo(File file);
    AudioInputStream decode(File file);
}

public class WavDecoder implements AudioDecoder { ... }
public class Mp3Decoder implements AudioDecoder { ... }
public class FlacDecoder implements AudioDecoder { ... }
```

#### 4.1.3 播放线程模型

- 播放线程：负责音频数据解码与输出
- 计时线程：负责更新播放进度
- UI线程：JavaFX Application Thread，负责界面更新

### 4.2 音频可视化实现

#### 4.2.1 频谱分析

使用 FFT（快速傅里叶变换）将时域音频数据转换为频域数据：

1. 从音频输出线路获取采样数据
2. 应用窗函数（汉宁窗）减少频谱泄漏
3. 执行 FFT 变换
4. 计算各频段能量
5. 平滑处理后渲染到界面

#### 4.2.2 渲染方式

使用 JavaFX Canvas 进行绘制，确保性能流畅。

### 4.3 歌词同步方案

#### 4.3.1 LRC 格式解析

LRC 格式示例：
```
[ti:歌曲名]
[ar:歌手名]
[al:专辑名]
[00:00.00]第一句歌词
[00:05.30]第二句歌词
```

#### 4.3.2 同步策略

- 维护当前歌词行索引
- 播放进度更新时，二分查找匹配的歌词行
- 平滑滚动动画效果

### 4.4 持久化方案

#### 4.4.1 数据存储位置

- Windows: `%APPDATA%/YinYuePlayer/`
- macOS: `~/Library/Application Support/YinYuePlayer/`
- Linux: `~/.config/YinYuePlayer/`

#### 4.4.2 存储内容

- 配置文件：config.json
- 播放列表：playlists/*.json
- 音乐库索引：library.json
- 缓存文件：cache/

## 五、开发规范

### 5.1 代码规范

- 遵循阿里巴巴Java开发手册
- 类名使用大驼峰命名法
- 方法名、变量名使用小驼峰命名法
- 常量使用全大写下划线分隔
- 包名使用全小写

### 5.2 注释规范

- 类注释：包含功能描述、作者、创建日期
- 方法注释：包含功能描述、参数说明、返回值说明
- 复杂逻辑必须添加行内注释

### 5.3 Git 提交规范

- feat: 新功能
- fix: 修复bug
- docs: 文档更新
- style: 代码格式调整
- refactor: 重构
- test: 测试相关
- chore: 构建/工具相关
